# 远程 Windows 电脑部署指南

## 前提条件

- Windows 11 + Docker Desktop 已安装并运行

## 步骤一：安装 Git（如果没有）

1. 下载 Git for Windows: https://git-scm.com/download/win
2. 安装时全部默认选项即可
3. 安装完成后，打开 **PowerShell**，验证：

```powershell
git --version
```

## 步骤二：克隆项目

打开 PowerShell，执行：

```powershell
# 选择一个目录，比如 D:\projects
mkdir D:\projects
cd D:\projects

# 克隆仓库
git clone https://github.com/shihanye0/stock_predicition.git

# 进入项目目录
cd stock_predicition
```

## 步骤三：配置环境变量

```powershell
# 复制环境变量模板
copy backend\.env.example backend\.env
```

然后用记事本编辑 `backend\.env`，按实际情况修改：

```powershell
notepad backend\.env
```

关键配置项（Docker 环境下一般不用改，默认即可）：

```env
DB_HOST=mysql
DB_PORT=3306
DB_USER=root
DB_PASSWORD=stock123
DB_NAME=stock_sentiment
REDIS_HOST=redis
REDIS_PORT=6379
```

## 步骤四：拷贝模型文件（重要）

BERT 模型文件 (409MB) 未上传到 GitHub，需要从本地拷贝：

**方法一：从本地电脑拷贝（推荐）**

将本地电脑的以下文件夹拷贝到远程电脑对应位置：

```
本地: E:\stock_predicition\backend\data\models\sentiment_model\
远程: D:\projects\stock_predicition\backend\data\models\sentiment_model\
```

确保远程电脑有这些文件：

```
backend/data/models/sentiment_model/
├── config.json
├── model.safetensors      ← 这是关键文件 (409MB)
├── special_tokens_map.json
├── tokenizer.json
├── tokenizer_config.json
└── vocab.txt
```

**方法二：自动下载**

如果不拷贝模型文件，系统首次启动时会自动从 Hugging Face 下载 `hfl/chinese-bert-wwm-ext`，但需要远程电脑能访问外网。

## 步骤五：启动 Docker 服务

确保 Docker Desktop 正在运行，然后执行：

```powershell
cd D:\projects\stock_predicition

# 首次启动（会构建镜像，需要几分钟）
docker-compose up -d

# 查看启动状态
docker-compose ps

# 查看日志（如有问题）
docker-compose logs -f backend
```

等待所有 6 个服务变为 `running` 状态：

| 服务 | 端口 | 说明 |
|------|------|------|
| stock-backend | 8000 | 后端 API |
| stock-frontend | 80 | 前端页面 |
| stock-mysql | 13306 | MySQL 数据库 |
| stock-redis | 6379 | Redis 缓存 |
| stock-prometheus | 19090 | 监控采集 |
| stock-grafana | 3000 | 监控面板 |

## 步骤六：验证部署

在浏览器中打开：

- 前端页面: http://localhost
- 后端 API 文档: http://localhost:8000/docs
- Grafana 监控: http://localhost:3000 (账号: admin / 密码: admin123)

## 常见问题

### Q: docker-compose up 报错 "port already in use"

某个端口被占用，修改 `docker-compose.yml` 中对应的端口映射，比如把 `"80:80"` 改为 `"8080:80"`。

### Q: 后端启动失败，日志显示数据库连接错误

等 MySQL 完全启动后再重启后端：

```powershell
docker-compose restart backend
```

### Q: 如何停止所有服务

```powershell
docker-compose down
```

### Q: 如何更新代码

```powershell
cd D:\projects\stock_predicition
git pull origin main
docker-compose up -d --build
```
