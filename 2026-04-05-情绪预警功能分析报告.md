# 情绪预警功能分析报告

> 日期: 2026-04-05  
> 模块: 情绪预警系统 (Alert Engine)

---

## 一、功能概述

情绪预警是本系统的**核心风控组件**，负责实时监测所有已追踪股票的情绪指标异常波动，当指标满足预设规则时自动生成预警记录并通知用户。

### 核心价值

| 维度 | 说明 |
|------|------|
| **风险预警** | 在市场情绪出现极端偏离时提前警示，辅助投资决策 |
| **异常检测** | 发现评论量暴增、情绪突变等异常行为，可能暗示重大事件 |
| **趋势反转信号** | 极端看涨/看跌指数往往是趋势反转的前兆 |
| **关注度监控** | 评论量暴增通常伴随重大利好/利空消息 |

---

## 二、现有规则（5条）

### 规则详情

| # | 规则名称 | 类型标识 | 检测逻辑 | 触发条件 |
|---|---------|---------|---------|---------|
| 1 | 情绪强度突变 | `intensity_spike` | 计算情绪强度的Z-Score | 当前值偏离历史均值 >= 2个标准差 |
| 2 | 看涨指数极端 | `bull_extreme` | 比较看涨指数与阈值 | 看涨指数 > 85% 或 < 15% |
| 3 | 看跌指数极端 | `bear_extreme` | 比较看跌指数与阈值 | 看跌指数 > 85% 或 < 15% |
| 4 | 评论量暴增 | `volume_surge` | 计算当日/历史均值比 | 单日评论量 >= 近期均值的3倍 |
| 5 | 情绪温度异常 | `temperature_anomaly` | 计算温度的Z-Score | 当前温度偏离历史均值 >= 2个标准差 |

### 严重程度判定

- **高危 (high)**: Z-Score >= 3、看涨/看跌指数 >= 90 或 <= 10、评论量 >= 5倍
- **中等 (medium)**: 达到基础触发条件但未达高危阈值
- **低危 (low)**: 预留，目前规则未产生低危预警

### 技术实现

- 引擎入口: `backend/app/services/alert/engine.py` - `AlertEngine` 类
- API路由: `backend/app/api/alerts.py` - 4个端点
- 前端页面: `frontend/src/views/Alerts.vue` - 统计卡片 + 筛选 + 列表 + 详情
- 回溯窗口: 默认30天历史数据，要求最少5个数据点

## 三、当前API端点

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/v1/alerts` | 获取预警列表（支持分页/筛选） |
| PUT | `/api/v1/alerts/{id}/read` | 标记单条已读 |
| PUT | `/api/v1/alerts/read-all` | 全部标记已读 |
| POST | `/api/v1/alerts/scan` | 手动触发预警扫描 |
| GET | `/api/v1/alerts/stats` | 预警统计（按严重程度/类型/股票） |

## 四、适用性扩展方向

以下是可以丰富系统功能的扩展方案，按优先级排列：

### 方向1: 自定义预警规则（高优先级）

**现状**: 5条规则的阈值是硬编码的（2σ、85/15、3倍）  
**扩展**: 允许用户在前端自定义预警规则

- 自定义阈值：用户可调整标准差倍数、极端值阈值、评论量倍数
- 新增规则类型：连续N天单边情绪、情绪一致性突变、跨股票联动异常
- 规则启停：按股票/按规则类型开关

**实现方式**: 新增 `alert_rules` 数据表 + Settings页面增加规则配置面板

---

### 方向2: 实时推送通知（高优先级）

**现状**: 预警只存入数据库，需要用户主动查看  
**扩展**: 多通道推送

| 通道 | 实现方案 |
|------|---------|
| WebSocket实时推送 | 后端建立WS连接，预警生成时立即推送到前端 |
| 邮件通知 | 高危预警触发时发送邮件摘要 |
| 企业微信/钉钉 | 通过Webhook推送到群组 |
| 浏览器通知 | 使用Web Notification API |

---

### 方向3: 预警与行情关联分析（中优先级）

**现状**: 预警只反映情绪面异常，不关联实际行情变化  
**扩展**: 

- 预警触发后自动拉取该股票的实时行情数据
- 统计预警后N天的实际涨跌幅，验证预警的有效性
- 生成"预警命中率"报告，帮助用户评估哪些规则最有价值
- 在预警详情中显示相关行情走势图

---

### 方向4: 智能预警等级（中优先级）

**现状**: 严重程度通过简单阈值判定  
**扩展**: 

- 引入机器学习模型，综合多个指标判断预警的真实严重性
- 历史预警回测：统计哪些预警对应了真实的股价波动
- 根据回测结果自动调整规则权重
- 降低误报率，提高预警质量

---

### 方向5: 板块/行业级预警（中优先级）

**现状**: 只支持单只股票级别的预警  
**扩展**: 

- 板块情绪预警：当整个行业板块情绪出现异常时触发
- 市场整体预警：跟踪大盘情绪指标
- 联动预警：同行业多只股票同时触发预警时升级为板块预警
- 舆情事件关联：将预警与新闻事件匹配

---

### 方向6: 预警Dashboard（低优先级）

**现状**: Alerts页面以列表为主  
**扩展**: 

- 预警热力图：按时间+股票维度可视化预警分布
- 预警趋势图：近期预警数量变化趋势
- TOP预警股票排行：哪些股票最频繁触发预警
- 规则效能分析：各规则的触发频率和命中率
