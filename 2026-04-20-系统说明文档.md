## 1. 系统概述

### 1.1 项目简介

基于Transformer的股票情绪预测系统是一个完整的全栈应用，用于分析中国A股市场投资者的情绪倾向。系统通过BERT深度学习模型对股吧、论坛等平台的评论进行情感分析，量化市场情绪指标，为投资决策提供数据支持。

### 1.2 核心功能

| 功能模块 | 描述 |
|---------|------|
| **多平台爬虫** | 支持东方财富、同花顺、新浪财经等平台评论采集 |
| **情感分析** | 基于BERT模型的三分类情感分析（积极/中性/消极） |
| **方面级分析** | 从业绩、政策、技术面、资金面、行业五个维度分析 |
| **情绪量化** | 计算看涨指数、看跌指数、情绪强度、温度、波动率 |
| **市场验证** | 分析情绪指标与股价的相关性和预测准确率 |
| **预警系统** | 自动检测情绪异常并生成预警 |
| **实时监控** | Dashboard展示市场情绪概览和热点股票 |

### 1.3 技术特点

- **高性能**: BERT模型GPU加速推理，支持批量处理
- **可扩展**: 微服务架构，Celery异步任务队列
- **高可用**: Redis缓存 + 数据库持久化
- **可观测**: Prometheus + Grafana监控
- **安全**: JWT认证 + 限流保护

## 5. API接口规范

### 5.1 股票相关接口

#### GET /api/v1/stocks
获取股票列表（分页）

**参数**:
- `page`: 页码 (默认1)
- `page_size`: 每页数量 (默认20, 最大100)
- `industry`: 行业筛选
- `market`: 市场筛选 (SH/SZ)
- `keyword`: 搜索关键词

**响应**:
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "stock_code": "000001",
      "stock_name": "平安银行",
      "industry": "银行",
      "market": "SZ"
    }
  ],
  "total": 1000,
  "page": 1,
  "page_size": 20
}
```

#### GET /api/v1/stocks/{code}
获取股票详情

#### GET /api/v1/stocks/{code}/comments
获取股票评论列表

**参数**:
- `platform`: 平台筛选
- `start_date`: 开始日期
- `end_date`: 结束日期

#### GET /api/v1/stocks/{code}/quotes
获取股票历史行情

#### GET /api/v1/stocks/{code}/emotion
获取股票情绪趋势

**参数**:
- `start_date`: 开始日期
- `end_date`: 结束日期
- `granularity`: 粒度 (hour/day)

#### GET /api/v1/stocks/{code}/aspects
获取方面级情感分析

**参数**:
- `limit`: 分析条数 (默认100)

#### GET /api/v1/stocks/{code}/profile
获取个股情绪画像（四维雷达图）

### 5.2 情感分析接口

#### POST /api/v1/sentiment/analyze
分析单条文本情感

**请求**:
```json
{
  "text": "今天这只股票涨停了，太牛了！"
}
```

**响应**:
```json
{
  "code": 200,
  "data": {
    "text": "今天这只股票涨停了，太牛了！",
    "label": "positive",
    "confidence": 0.95,
    "scores": {
      "positive": 0.95,
      "neutral": 0.03,
      "negative": 0.02
    }
  }
}
```

#### POST /api/v1/sentiment/batch
批量分析文本情感（最多100条）

#### GET /api/v1/sentiment/stats

## 6. 核心模块详解

### 6.1 情感分析服务

#### BERT情感分析器 (SentimentAnalyzer)

**位置**: `backend/app/services/sentiment/analyzer.py`

**功能**: 基于BERT模型进行三分类情感分析

**核心方法**:

| 方法 | 说明 |
|------|------|
| `analyze(text)` | 分析单条文本情感 |
| `analyze_batch(texts, batch_size)` | 批量分析（最大100条） |
| `_preprocess(text)` | 文本预处理 |
| `_predict(texts)` | 模型推理 |

**模型配置**:
- 默认模型: `hfl/chinese-bert-wwm-ext`
- 最大长度: 128 tokens
- 标签映射: {0: negative, 1: neutral, 2: positive}

**使用示例**:
```python
from app.services.sentiment.analyzer import get_analyzer

analyzer = get_analyzer()
result = analyzer.analyze("今天股票涨了很开心")
# result: SentimentResult(text="...", label="positive", confidence=0.95, scores={...})
```

#### 方面级情感分析器 (AspectSentimentAnalyzer)

**位置**: `backend/app/services/sentiment/aspect_analyzer.py`

**功能**: 从五个维度分析评论的情感倾向

**方面定义**:

| 方面 | 关键词示例 |
|------|-----------|
| 业绩 | 营收、利润、财报、分红、ROE |
| 政策 | 降准、降息、监管、改革、补贴 |
| 技术面 | 均线、MACD、突破、金叉、支撑 |
| 资金面 | 主力、北向、融资、龙虎榜 |
| 行业 | 景气度、龙头、赛道、竞争 |

**核心方法**:

| 方法 | 说明 |
|------|------|
| `analyze(text)` | 分析单条文本的方面级情感 |
| `analyze_batch(texts)` | 批量分析 |
| `get_stock_aspect_summary(results)` | 汇总多文本的方面分布 |

**使用示例**:
```python
from app.services.sentiment.aspect_analyzer import get_aspect_analyzer

analyzer = get_aspect_analyzer()
result = analyzer.analyze("茅台业绩超预期，股价创新高")
# result.aspects: [{aspect: "业绩", sentiment_label: "positive", ...}, ...]
```

#### 金融词典 (FinanceLexicon)

**位置**: `backend/app/services/sentiment/lexicon.py`

**功能**: 包含800+金融领域情感词汇

**词库结构**:
```python
LEXICON = {
    "positive": ["涨停", "利好", "大涨", "突破", "净买入", ...],
    "negative": ["跌停", "利空", "大跌", "破位", "净卖出", ...],
    "intensity": ["暴涨", "狂跌", ...]  # 程度词
}
```

### 6.2 爬虫服务

#### 爬虫基类 (BaseCrawler)

**位置**: `backend/app/services/crawler/base.py`

**反爬策略**:
- 随机User-Agent轮换
- 请求间隔3-8秒随机延迟
- 最大重试3次（指数退避）
- 超时30秒

**核心方法**:

| `fetch(url, params, retry)` | 发起HTTP请求 |
| `fetch_json(url, params)` | 获取JSON响应 |
| `crawl_stock(code, start_date, end_date)` | 爬取单只股票 |
| `crawl(codes, callback)` | 批量爬取 |
| `stop()` | 停止爬虫 |

#### 爬虫管理器 (CrawlerManager)

**位置**: `backend/app/services/crawler/manager.py`

**功能**: 统一管理多个平台爬虫

**支持的平台**:

**核心方法**:

| `get_status(platform)` | 获取爬虫状态 |
| `get_stats()` | 获取统计数据 |

### 6.3 情绪量化服务

#### 情绪计算器 (EmotionCalculator)

**位置**: `backend/app/services/quantify/calculator.py`

**量化指标**:

| 指标 | 计算公式 | 范围 |
|------|---------|------|
| 看涨指数 | 积极评论数 / 总评论数 × 100 | 0-100 |
| 看跌指数 | 消极评论数 / 总评论数 × 100 | 0-100 |
| 情绪强度 | (积极数 - 消极数) / 总数 | -1到1 |
| 情绪温度 | 置信度加权情绪分 | 0-100 |
| 情绪波动 | 历史强度标准差 | 0-1 |

**核心方法**:

| 方法 | 说明 |
|------|------|
| `calculate_bull_index(pos, total)` | 计算看涨指数 |
| `calculate_bear_index(neg, total)` | 计算看跌指数 |
| `calculate_intensity(pos, neg, total)` | 计算情绪强度 |
| `calculate_temperature(sentiments, total)` | 计算情绪温度 |
| `calculate_volatility(intensities)` | 计算波动率 |
| `calculate_metrics(sentiments)` | 计算所有指标 |

#### 情绪画像 (EmotionProfile)

**位置**: `backend/app/services/quantify/profiler.py`

**四维画像**:

| 维度 | 说明 |
|------|------|
| 波动性 | 情绪稳定程度 |
| 偏向性 | 多空倾向 |
| 活跃度 | 评论量/关注度 |
| 一致性 | 观点集中度 |

### 6.4 预警引擎

#### 预警规则 (AlertEngine)

**位置**: `backend/app/services/alert/engine.py`

**检测规则**:

| 规则 | 条件 | 严重程度 |
|------|------|---------|
| 情绪强度突变 | 偏离历史均值>2σ | high/medium |
| 看涨指数极端 | >85 或 <15 | high/medium |
| 看跌指数极端 | >85 或 <15 | high/medium |
| 评论量暴增 | >均值3倍 | high/medium |
| 情绪温度异常 | 偏离历史均值>2σ | medium/low |

**核心方法**:

| 方法 | 说明 |
|------|------|
| `scan_stock(code, lookback_days)` | 扫描单只股票 |
| `scan_all_stocks()` | 扫描所有股票 |
| `_check_std_deviation()` | 检查标准差偏离 |
| `_check_extreme()` | 检查极端值 |
| `_check_volume_surge()` | 检查量能暴增 |

### 6.5 市场验证服务

#### 市场验证器 (MarketValidator)

**位置**: `backend/app/services/market/validator.py`

**验证指标**:

| 功能 | 方法 | 说明 |
|------|------|------|
| Pearson相关 | `calculate_correlation()` | 线性相关 |
| Spearman相关 | `calculate_spearman()` | 秩相关 |
| 方向准确率 | `analyze_direction_accuracy()` | 涨跌预测准确率 |
| 最优领先天数 | `find_optimal_lag()` | 寻找最佳lag |
| 格兰杰因果 | `granger_causality_test()` | 因果检验 |
| 完整报告 | `generate_validation_report()` | 综合报告 |

### 6.6 文本处理服务

#### 文本处理器 (TextProcessor)

**位置**: `backend/app/services/processor/text_processor.py`

**处理流程**:

1. 清除HTML标签
2. 清除URL
3. 处理表情符号 [xxx]
4. 清除@用户
5. 清除#话题#
6. 清除特殊字符
7. 规范化空白

**核心方法**:

| 方法 | 说明 |
|------|------|
| `clean(text)` | 完整清洗流程 |
| `tokenize(text, remove_stopwords)` | 中文分词 (Jieba) |
| `extract_keywords(text, topk)` | 关键词提取 (TF-IDF) |
| `extract_stock_codes(text)` | 股票代码提取 |
| `compute_hash(text)` | 文本去重hash |
| `is_similar(text1, text2, threshold)` | 文本相似度判断 |
